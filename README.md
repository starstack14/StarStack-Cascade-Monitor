# StarStack Cascade Monitor

Windows-виджет для наблюдения за каскадом Remnawave, роутером NX31/OpenWrt и домашней сетью. Приложение работает в системном трее, собирается в один EXE и предоставляет защищённую мобильную web-панель.

Текущая схема:

```text
Домашние устройства → NX31 → Moscow (VLESS Reality TCP) → Germany (XHTTP)
                         └─ direct для выбранных доменов и локальных сетей
```

## Основные возможности

### Каскад и Remnawave

- состояние Moscow и Germany, TCP-задержка, CPU/Load, RAM и число пользователей;
- общий трафик нод и приблизительная текущая скорость;
- активные пользователи, нода, устройство, ОС и IP при наличии данных HWID;
- наглядная схема `NX31 → Moscow → Germany`;
- проверка выходных IP Direct, Moscow и Germany;
- тест скорости выбранной ноды через контекстное меню;
- уведомления при падении и восстановлении нод, превышении RAM или задержки.

### NX31 и sing-box

- мониторинг OpenWrt через отдельный SSH Ed25519-ключ;
- WAN-IP, интерфейс, uptime, Load Average, RAM и состояние sing-box;
- просмотр журнала и ручной перезапуск sing-box;
- перезагрузка роутера только после подтверждения;
- безопасный watchdog: 3 сбоя подряд, пауза 10 минут, не более 2 перезапусков в час;
- диагностический снимок процессов, маршрутов, DNS и журнала перед watchdog-перезапуском;
- DNS/IPv6 leak test;
- прямые исключения для доменов настраиваются в клиентском route-правиле NX31.

### Домашняя сеть

- LAN/Wi-Fi-устройства с IP, MAC, диапазоном, уровнем сигнала и скоростью линка;
- текущая скорость и расход трафика за день/месяц через `nlbwmon`;
- обнаружение новых MAC-адресов и системные уведомления;
- собственные имена и отметка доверенных устройств;
- история появления и отключения;
- графики трафика за 24 часа и 7 дней;
- подтверждаемая блокировка и разблокировка устройства правилами firewall NX31;
- защита от блокировки компьютера, на котором запущен монитор.

### История и резервные копии

- SQLite-история показателей, событий, устройств и трафика;
- журнал событий за 7 дней;
- ручные и еженедельные автоматические копии;
- хранение последних 5 архивов;
- в архив входят OpenWrt, sing-box, настройки приложения и история;
- архивы ограничиваются Windows ACL текущего пользователя и SYSTEM.

### Мобильная web-панель

- собственный домен задаётся в настройках приложения;
- адаптивный интерфейс для телефона;
- состояние каскада, маршрутов, нод, VPN-пользователей и домашней сети;
- отдельные логин и пароль, пароль хранится через Windows DPAPI;
- cookie-сессии `Secure`, `HttpOnly`, `SameSite=Strict`;
- ограничение попыток входа и защитные HTTP-заголовки;
- API работает только после авторизации и доступен только для чтения;
- backend слушает исключительно `127.0.0.1:8765`;
- Caddy завершает TLS и автоматически управляет сертификатом Let’s Encrypt;
- web-панель доступна, пока приложение запущено или находится в трее.

Публичный маршрут web-панели:

```text
Телефон → HTTPS 443 → OpenWrt DNAT → Caddy на ПК → 127.0.0.1:8765 → Monitor
```

## Интерфейс

- крестик скрывает окно в трей, не завершая приложение;
- двойной щелчок по значку возвращает окно;
- для полного завершения используется `Трей → Выход`;
- поддерживаются компактный режим, окно поверх остальных и регулируемая прозрачность;
- период обновления настраивается от 5 до 300 секунд;
- повторный запуск блокируется системным mutex.

## Требования

- Windows 10/11 и Python 3.11 для запуска из исходников;
- Remnawave Panel и отдельный API-токен с правами чтения;
- OpenWrt с SSH-доступом по ключу для функций роутера;
- `nlbwmon` для статистики трафика по домашним устройствам;
- Caddy, публичный DNS и перенаправление 80/443 — только если нужна мобильная HTTPS-панель.

## Быстрый запуск

1. Клонируйте репозиторий.
2. Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

3. Создайте отдельный ключ командой `.\generate_router_key.ps1` и добавьте public key в OpenWrt.
4. Запустите `start_widget_console.bat` и заполните настройки.
5. Для мобильной панели выполните `.\install_caddy.ps1`, настройте DNS и безопасный DNAT TCP 80/443 на компьютер с монитором.

Не копируйте `config.example.json` поверх рабочего конфига с секретами. Реальный `config.local.json` приложение создаёт и обновляет локально.

## Запуск готовой сборки

Готовая сборка находится здесь:

```text
dist\StarStack-Cascade-Monitor.exe
```

Также можно запустить `start_widget.bat` — он автоматически выберет EXE. Для диагностики исходной Python-версии используется `start_widget_console.bat`.

При первом запуске в настройках указываются:

1. URL панели Remnawave.
2. Отдельный API-токен с минимальными правами чтения Nodes, Users и HWID Devices.
3. Адрес NX31 и путь к SSH-ключу.
4. Отдельные реквизиты мобильной web-панели.

## Автозапуск

Запустите `enable_autostart.ps1` через PowerShell. Для удаления автозапуска используйте `disable_autostart.ps1`.

## Настройка OpenWrt

Минимальная настройка зависит от конкретной прошивки и адреса LAN. Перед изменением firewall сделайте системную резервную копию OpenWrt.

- установите и включите `nlbwmon` для трафика устройств;
- закрепите DHCP-адрес компьютера мониторинга, если публикуете web-панель;
- перенаправляйте 80/443 только на Caddy, не на локальный backend `8765`;
- разрешите Caddy во входящем Windows Firewall;
- проверьте конфигурацию firewall перед применением;
- не публикуйте LuCI и SSH в интернет.

Приложение ожидает ноды, содержащие в имени `Moscow`/`RU` и `Germany`/`DE`. При другой схеме адаптируйте `node_country_and_name()` и проверку маршрутов.

## Сборка EXE

Требуется Python 3.11. Скрипт сам подготовит зависимости и PyInstaller:

```powershell
.\build_exe.ps1
```

Результат появится в `dist\StarStack-Cascade-Monitor.exe`.

GitHub Actions также проверяет Python-файлы и собирает Windows EXE для каждого push/PR в `main`; готовая сборка доступна как workflow artifact.

## Структура проекта

```text
main.py          основной Tkinter-интерфейс и оркестрация
webpanel.py      локальный read-only HTTP backend и управление Caddy
openwrt.py       SSH/LuCI-клиенты NX31, устройства, трафик и управление
remnawave.py     клиент Remnawave API
history.py       SQLite-история и события
routecheck.py    проверка выходных IP
leaktest.py      DNS/IPv6-проверка
nettest.py       тест скорости
security.py      Windows DPAPI
Caddyfile        HTTPS reverse proxy
keys\            SSH-ключ и known_hosts
backups\         защищённые резервные копии
diagnostics\     снимки watchdog
logs\            журнал Caddy
tools\caddy.exe  локально установленный HTTPS reverse proxy (не хранится в Git)
```

## Безопасность

- не используйте Superadmin JWT; создавайте отдельный API-токен с минимальным read-only scope;
- API-токен, служебные параметры, пароль роутера и пароль web-панели хранятся через DPAPI;
- `config.local.json`, private SSH key, резервные копии и диагностика не должны попадать в Git;
- backend web-панели не принимает подключения из LAN/WAN напрямую;
- действия, меняющие состояние NX31, требуют подтверждения, кроме ограниченного watchdog sing-box;
- после публикации токена или пароля в сообщениях его следует отозвать и заменить;
- перед восстановлением OpenWrt проверяйте архив и используйте локальный доступ к роутеру.
